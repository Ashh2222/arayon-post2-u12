# ADR-001: Arquitectura Hexagonal para aislar el dominio

## Estado

Aceptado — 2026-05-17

---

## Contexto

El sistema de pedidos debe soportar múltiples tipos de procesamiento y canales de notificación.

El acoplamiento directo entre dominio e infraestructura dificulta las pruebas unitarias y hace costoso reemplazar tecnologías de persistencia.

---

## Decisión

Se adopta arquitectura hexagonal.

El dominio define puertos mediante interfaces y las capas externas implementan dichos contratos.

El dominio no depende de Spring ni de infraestructura.

---

## Consecuencias

### Positivas

- Dominio desacoplado
- Mayor testeabilidad
- Facilidad para cambiar persistencia

### Negativas

- Mayor número de clases
- Curva de aprendizaje más alta
