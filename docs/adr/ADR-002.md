# ADR-002: Factory + Strategy para procesamiento de pedidos

## Estado

Aceptado — 2026-05-17

---

## Contexto

El sistema posee múltiples tipos de pedido con reglas distintas de cálculo.

El uso de condicionales extensos viola Open/Closed Principle.

---

## Decisión

Se implementa Strategy para encapsular algoritmos independientes y Factory para seleccionar dinámicamente la estrategia correcta.

---

## Consecuencias

### Positivas

- Fácil extensión
- Menor acoplamiento
- Procesadores independientes

### Negativas

- Mayor cantidad de clases
- Complejidad inicial más alta
