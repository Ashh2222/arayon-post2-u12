# ADR-003: Observer mediante Spring Events

## Estado

Aceptado — 2026-05-17

---

## Contexto

El sistema debe notificar múltiples canales cuando un pedido es procesado.

El acoplamiento directo a email y logging hace frágil la lógica principal.

---

## Decisión

Se implementa Observer utilizando ApplicationEventPublisher y listeners con @EventListener.

---

## Consecuencias

### Positivas

- Desacoplamiento
- Escalabilidad
- Nuevos listeners sin modificar lógica principal

### Negativas

- Debugging más complejo
- Orden de ejecución no garantizado
